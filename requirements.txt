# DO NOT BUMP CASUALLY - this version is tied to the splink salting bug.
# With splink 4.0.16 + duckdb 1.5.4, estimate_u_using_random_sampling() raises
# "Salting partitions must be specified". Work around it by skipping random-
# sampling u-estimation and using estimate_probability_two_random_records_match()
# with deterministic rules, followed by EM training.
# Do NOT downgrade duckdb to 1.4.0 / 1.3.2 to dodge this - that path introduces
# other type-incompatibility problems.

# already CPU-only (torch.__version__ then reports 2.13.0+cpu, which is normal).
# For an explicit CPU build on Linux:
#   pip install torch==2.13.0 --index-url https://download.pytorch.org/whl/cpu
